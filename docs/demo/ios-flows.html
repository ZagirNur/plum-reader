Plum Reader · iOS app flows

Как iOS-приложение работает с API

Карта пользовательских сценариев + точные API-вызовы в каждой точке. Каждый телефон — отдельный экран; стрелка — переход. Под каждым экраном — что приложение в этот момент дёргает на бэкенде. Авторитативный контракт — openapi.yaml, реальные ответы сервера — sample-output.txt.

AuthJWT (HS256, 60 мин). На каждом authed-запросе — Authorization: Bearer …. На 401 → re-login.
PollingПосле upload фронт опрашивает GET /books/{id} каждые ~1с пока status / markupStatus ≠ ready.
SandboxHTML страниц рендерится в WKWebView с отключённым JS — EPUB-content неприоверенный.
ErrorsСтабильный enum error в JSON. См. errors.md.

После выхода
Очищаем JWT из Keychain, чистим in-memory cache (URLCache.shared.removeAllCachedResponses), возвращаемся на splash.
Серверный logout не нужен (stateless JWT).
В roadmap — refresh-token revocation list.
Гайд

Что нужно агенту интеграции iOS

Базовое

  • JWT хранить в Keychain (`kSecClassGenericPassword`), не в UserDefaults.
  • Bearer header на каждый authed-запрос.
  • URLSession interceptor: на 401 → удалить токен → redirect /login.
  • Pre-flight CORS — не наш кейс (нативное приложение), но если будет WKWebView для рендера — обращаемся напрямую к сокету, без CORS.

Polling

  • После upload → polling `GET /books/{id}` каждые 1с.
  • Стоп условие: `status="ready"` (для CTA "Читать") и/или `markupStatus="ready"` (для CTA "Словарь").
  • Терминальное: `status="failed"` или `markupStatus="failed"` → показать error-экран.
  • Timeout polling'а — 60с. Дальше — toast "обновим позже".

Reader / pages

  • `xhtml` — raw spine document, рендерить в `WKWebView` с `configuration.preferences.javaScriptEnabled = false` и `navigationDelegate` который запрещает любую навигацию.
  • Кастомный CSS поверх (typography, dark mode, размер шрифта) — инжектится через `userContentController`.
  • prevIdx/nextIdx === null → disable swipe в эту сторону.

Прогресс

  • PATCH `/progress` с debounce 1-2 секунды после последнего перелистывания.
  • На background/foreground — flush pending PATCH сразу.
  • `updatedAt` хранить локально; при multi-device конфликте — last-write-wins по `updatedAt`.

Кэш

  • BookSummary в SQLite/Core Data — offline-доступ к библиотеке.
  • Pages в файловом кэше — после первого чтения не лезть в API.
  • Vocabulary иммутабелен на (bookId, markupStatus=ready) → кэшировать "навсегда".
